Pruebas Code Injection

Testea inyección de código JavaScript malicioso para validación de protección de WAF

Prueba Code Injection (JavaScript)

Simula inyección de código JavaScript malicioso:

Pruebas peligrosas: process.exit(), require('fs'), global.process

Ejemplos de Código Malicioso

Inyección de Código JavaScript (Node.js):

  • process.exit() - Terminar el servidor
  • require('fs').readFileSync('/etc/passwd') - Leer archivos del sistema
  • require('child_process').execSync('ls -la') - Ejecutar comandos del sistema
  • global.process.env - Acceder a variables de entorno
  • require('os').userInfo() - Información del usuario del sistema
  • require('fs').writeFileSync('hack.txt', 'hacked') - Escribir archivos

Protecciones WAF Esperadas

El WAF debería bloquear:

  • Palabras clave de ejecución: exec, eval, system
  • Acceso a módulos sensibles: require, import, fs, child_process
  • Path traversal: ../, ..\\
