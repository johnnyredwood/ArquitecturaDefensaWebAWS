Prueba de Búsqueda - SQL Injection

Simula vulnerabilidades SQL Injection en formularios de búsqueda

Formulario de Búsqueda

Ingresa consultas de búsqueda para probar la protección del WAF contra SQL Injection:

Ejemplos para Probar

Pruebas de SQL Injection:

  • ' OR '1'='1 - Retorna todos los registros
  • '; DROP TABLE users-- - Inyección destructiva
  • ' UNION SELECT username, password FROM users-- - Extracción de datos
  • admin' -- - Bypass de autenticación
  • ' AND 1=1 -- - Condición siempre verdadera

Búsquedas legítimas:

  • Producto - Búsqueda normal
  • electronics - Buscar por categoría
Nota: El WAF debería bloquear las consultas maliciosas mientras permite las legítimas.

Resultados Esperados

Comportamiento normal: Búsquedas legítimas muestran resultados filtrados
Comportamiento bloqueado: Consultas SQL Injection son bloqueadas por el WAF
Comportamiento vulnerable: Si el WAF no está configurado, las consultas maliciosas podrían ejecutarse
